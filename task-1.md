**Также есть замечания к структуре требований:** требования приведены единым
списком без группировки по функциональным блокам: лимиты корзины, добавление
товара, изменение количества, удаление, отображение корзины, цены и реклама
описаны вперемешку. Из-за этого требования сложнее анализировать, проверять на
полноту и использовать при разработке.

**Дополнительно - отсутствует единообразие терминологии:** в разных пунктах
используются термины “товар”, “продукт”, “единица товара”, “позиция”. Для
исключения разночтений стоит ввести глоссарий терминов и явно определить, что
означает каждый термин. Например: “товар” — карточка товара из каталога,
“товарная позиция” — строка корзины по конкретному товару, “единица товара” —
количество товара внутри товарной позиции.

### 2. Исправленная версия требований

#### 2.1. Глоссарий

| Термин           | Описание                                                              |
|------------------|-----------------------------------------------------------------------|
| Товар            | Карточка товара из каталога интернет-магазина.                        |
| Товарная позиция | Строка корзины, соответствующая одному конкретному товару.            |
| Единица товара   | Количество товара внутри товарной позиции.                            |
| Корзина          | Набор товарных позиций, выбранных пользователем до оформления заказа. |

#### 2.2. Функциональные требования

###### _Лимиты корзины_

**CART-LIMIT-001.** Корзина пользователя может содержать не более 5
различных товарных позиций. <br>
**CART-LIMIT-002.** Общее количество единиц товаров в корзине пользователя
не должно превышать 20.<br>
**CART-LIMIT-003.** Одна товарная позиция в корзине пользователя может содержать
от 1 до 10 единиц товара включительно.

###### _Добавление товара в корзину_

**CART-ADD-001.** Пользователь может добавить товар в корзину со страницы товара
или из каталога.<br>
**CART-ADD-002.** При добавлении товара, которого еще нет в корзине, система
создает новую товарную позицию.<br>
**CART-ADD-003.** При повторном добавлении товара, который уже есть в корзине,
система увеличивает количество существующей товарной позиции.<br>
**CART-ADD-004.** Система не должна добавлять товар в корзину, если после
добавления будет превышен хотя бы один из лимитов, описанных в CART-LIMIT-001,
CART-LIMIT-002 или CART-LIMIT-003.

###### _Изменение количества товара_

**CART-QTY-001.** Пользователь может изменить количество единиц товара в
товарной позиции в пределах от 1 до 10 включительно.<br>
**CART-QTY-002.** Изменение количества товара выполняется с помощью кнопок “+”
и “-”.<br>
**CART-QTY-003.** Кнопка “-” недоступна, если количество единиц товара в
товарной позиции равно 1.<br>
**CART-QTY-004.** Если при изменении количества будет превышен общий лимит
корзины в 20 единиц товаров, система не должна применять изменение.

###### _Удаление товара из корзины_

**CART-DELETE-001.** Для каждой товарной позиции в корзине отображается кнопка
“Удалить”.<br>
**CART-DELETE-002.** При нажатии на кнопку “Удалить” система удаляет товарную
позицию из корзины.<br>
**CART-DELETE-003.** После удаления товарной позиции система пересчитывает
итоговую стоимость корзины.

###### _Сообщения об ошибках и ограничениях_

**CART-VALIDATION-001.** Если пользователь пытается добавить больше 10 единиц
одного товара, система не добавляет товар и показывает сообщение: “В корзину
можно добавить не более 10 единиц одного товара”.<br>
**CART-VALIDATION-002.** Если пользователь пытается добавить больше 5 различных 
товарных позиций, система не добавляет товар и показывает сообщение: “В корзине
может быть не более 5 разных товаров”.<br>
**CART-VALIDATION-003.** Если пользователь пытается добавить товар сверх общего
лимита в 20 единиц товаров, система не добавляет товар и показывает сообщение:
“Общее количество товаров в корзине не может превышать 20”.

###### _Отображение страницы корзины_

**CART-VIEW-001.** На странице корзины отображается список товарных позиций.<br>
**CART-VIEW-002.** Для каждой товарной позиции отображаются: наименование
товара, количество единиц товара, цена за единицу товара, общая стоимость
товарной позиции и кнопка “Удалить”.<br>
**CART-VIEW-003.** Общая стоимость товарной позиции рассчитывается по формуле:
количество единиц товара × цена за единицу товара.<br>
**CART-VIEW-004.** На странице корзины отображается итоговая стоимость всех
товарных позиций.<br>
**CART-VIEW-005.** На странице корзины отображается кнопка “Оформить заказ”.

###### _Цена товара в корзине_

**CART-PRICE-001.** Цена товара в корзине должна соответствовать актуальной цене
товара в каталоге.<br>
**CART-PRICE-002.** Если цена товара изменилась в каталоге, система
автоматически обновляет цену соответствующей товарной позиции в корзине
пользователя.<br>
**CART-PRICE-003.** Финальная цена товара фиксируется в момент оформления
заказа.

###### _Рекламный блок в корзине_

**CART-ADS-001.** В корзине может отображаться рекламный блок с товарами из
каталога.<br>
**CART-ADS-002.** Рекламный блок отображается на странице корзины под списком
товарных позиций.<br>
**CART-ADS-003.** В рекламном блоке отображается не более 3 товаров.<br>
**CART-ADS-004.** Рекламный блок отображается по будним дням в следующие
временные интервалы: с 09:00 до 12:00 и с 18:00 до 21:00 по московскому времени.<br>
**CART-ADS-005.** В выходные дни рекламный блок не отображается.<br>
**CART-ADS-006.** Товары для рекламного блока выбираются из активных товаров
каталога, доступных для покупки.<br>
**CART-ADS-007.** Товары, которые уже находятся в корзине пользователя, не
должны отображаться в рекламном блоке.

#### 2.3. Принятые допущения

В исправленной версии требований выбран следующий вариант бизнес-логики:

* удаление товарной позиции выполняется только через отдельную кнопку “Удалить”;
* изменение количества товара доступно в диапазоне от 1 до 10 включительно;
* цена товара в корзине актуализируется при изменении цены в каталоге;
* финальная цена фиксируется в момент оформления заказа;
* рекламный блок описан как часть страницы корзины с конкретными правилами
  показа.

Требования сгруппированы по функциональным разделам. Идентификаторы
требований сформированы по шаблону `CART-{SECTION}-{NUMBER}`, нумерация ведется
отдельно внутри каждого раздела.
