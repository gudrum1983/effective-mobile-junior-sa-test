# Тестовое задание Junior Системный аналитик

Репозиторий содержит выполненное тестовое задание на позицию Junior системного
аналитика.

## Формат решения

Все задания оформлены в формате Markdown: текстовые описания, таблицы, описание
API и архитектурные схемы.

Так как по условию тестовое задание необходимо предоставить в виде ссылки на
GitHub, Markdown выбран как основной формат решения. Он удобно отображается
непосредственно в GitHub и не требует скачивания файлов или открытия решения
в сторонних приложениях.

Для удобства все решения из файлов `*.md` также объединены в единый PDF-файл.

## Выполненные задания

### Задание 1. Анализ требований

Проведен анализ фрагмента технического задания на функционал корзины
интернет-магазина.

В решении:

* найдены логические противоречия, недочеты и неполные места в требованиях;
* предложена исправленная версия требований;
* подготовлены уточняющие вопросы к продукт-менеджеру или бизнес-заказчику.

[Открыть решение](task-1.md)

### Задание 2. Проектирование REST API

Спроектирован REST API для мобильного экрана выбора магазина-партнера.

В решении:

* описан REST API запрос, который вызывается при переходе на экран;
* приведен пример JSON-ответа в соответствии с макетом;
* описаны основные поля ответа и возможные ошибки;
* указаны принятые допущения и возможное расширение API.

[Открыть решение](task-2.md)

### Задание 3. Архитектура PUSH-уведомлений

Подготовлена верхнеуровневая архитектура отправки PUSH-уведомлений
в мобильное приложение при микросервисной архитектуре backend.

В решении:

* описана регистрация push-токена устройства;
* построена схема отправки PUSH-уведомлений по событиям из микросервисов;
* указаны принятые допущения по работе отложенных и периодических событий.

[Открыть решение](task-3.md)
